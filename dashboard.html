Caching Strategies
cache-aside

Cache-Aside

App checks Redis first. On miss, loads from PostgreSQL and stores in Redis. Best for read-heavy workloads.

write-through

Write-Through

Every write goes to both DB and cache synchronously. Strong consistency with slight write overhead.

write-back

Write-Back

Writes hit cache first; DB is updated lazily. Maximum write throughput with small durability tradeoff.
